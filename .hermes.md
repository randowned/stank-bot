# StankBot

Hermes working guidelines when operating in this repository.

## Project Identity
- **Main project**: stank-bot — a Discord bot tracking "Stank" sticker chains with a web dashboard.
- **Other projects**: may exist under `/home/randowned/workspace/` — check for `.hermes.md` there too.

## Workflow Rules
- Follow the conventions already in `AGENTS.md` — it's the source of truth for this repo.
- Prefer editing existing code over adding new files, unless a new module is genuinely needed.
- Do not invent abstractions (base classes, plugin interfaces, DI frameworks) until a second concrete implementation forces the shape.

## Tech Stack
- Python 3.12 with `uv` package manager
- Discord.py for bot, SvelteKit for frontend (SPA mode, no SSR)
- SQLite via aiosqlite + SQLAlchemy + Alembic migrations
- Event-sourced architecture — every SP change is an immutable event
- Railway deployment (Docker)

## Environment
- Local dev: `ENV=dev-mock` (no Discord credentials needed)
- Real Discord testing: `ENV=dev`
- Prod: Railway env vars

## Commands
- Dev server: `./scripts/dev.sh` (Linux/macOS) or `./scripts/dev.ps1` (Windows)
- Tests: `pytest`
- Lint: `ruff check src tests`
- Typecheck: `mypy src`
- DB migrations: `alembic upgrade head`

## Kanban
- Tasks tracked via `hermes kanban` or `/kanban` slash command
- Board: `~/.hermes/kanban.db`
